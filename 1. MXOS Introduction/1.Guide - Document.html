Guide

Welcome to the MXOS handbook. This manual will guide you step by step to explore the development of MXOS-based IoT devices.

What is MXOS

MXOS a microcontroller-based Internet access operating system, an ideal software platform for developing IoT devices.

MXOS enables your application code to be understood and executed by the device through a hierarchical, componentized design. MXOS's application layer code is written in C/C++ and can be manipulated by calling the API functions provided by MXOS to implement system calls. These API functions provided by MXOS allow your code to run on different MCU platforms in a uniform manner.

This unified layered design dramatically reduces the challenges of integrating different MCUs and a large number of intermediate software. The componentized design can easily realize the mutual integration of various software functions and realize complex IoT applications quickly.

Learn more about MXOS.

MXOS's operational carrier - MCU, module and development board MXOS is an IoT operating system running on MCU microcontrollers. The carrier is an MCU chip. These MCU chips are combined with wireless communication technology to become a module running MXOS. These chips and modules are installed on the device and become the brain of IoT devices. In order to achieve rapid prototyping, we developed the corresponding MXOSKit development board for each module, and also developed an expansion board based on the original development board in cooperation with the original MCU chip. At development time, you can compile, debug and functionally test MXOS applications on the MXOS development board.

Learn more about modules and boards that run MXOS.

Where do you start?

Developers should first install the development tools provided by MXOS. You can verify that the entire MXOS development environment is properly installed with one of the simplest MXOS applications: Helloworld. Once you have mastered how to compile an existing application, you can start writing your own application.

Development tools

  • MXOS Cube：It allows you to get the latest version of the code from the code base and complete the compilation, download and debugging of the MXOS application.
  • MiCoder IDE ：Allow developers to import code into the IDE environment for compilation, download, and debugging for secondary development.
  • Simulation debugger：It is an essential hardware tool for application downloading and debugging.

How to continue next step
