Bootloader

Bootloader is a small program that runs before the operating system kernel or user application runs. Through this small program, we can initialize the hardware device and create a map of the memory space to bring the system's hardware and software environment to a suitable state to prepare the correct environment for the final call to the operating system kernel or user application.

This document contains a list of commands for the MXOS Bootloader mode, the steps to go and the specific command usage.

Function list

MXOS's Bootloader mode provides users with the following commands: Name Function

Command code Name Function
0 BOOTUPDATA Update bootloader firmware
1 FWUPDATA Update application application firmware
2 DRIVERUPDATE Update RF RF drive
3 PARAUPDATE Update MXOS setup parameters
4 FLASHUPDATE Update flash content
5 MEMORYMAP List flash memory allocation map
6 BOOT Running the application
7 REBOOT Restart

Among them:

  • 1.bootloader firmware, compile command in MXOS Cube: MXOS make sub_build.bootloader@MK3165@NoRTOS, compile the .bin file in build//binary.
  • 2.application application firmware, the compile directive in MXOS Cbue is: helloworld@MK3165 or helloworld@MK3080B@MOC. It can be found in build//binary after compiling, such as xxxx@MK3165.bin or xxx@MK3080B@MOC.ota.bin in the build/xxx/binary folder.
    1. In addition, all flash firmware, namely: @MKxxx.bin or xxx.all.bin, including: bootloader, application and RF firmware, compiler command in MXOS Cbue: helloworld@MK3165 total or helloworld@MK3080B@MOC total, compiled can be found in build//binary, such as ’xxxx@MK3165.all.bin‘ or ’xxx@MK3080B@MOC.all.bin‘.
  • 4.RF RF firmware in mxos: \mxos-os\resources\wifi_firmware, where:
    • 43362: Support models EMW3162, 3165, 3166.
    • 43438: Support model EMW3239.
    • mw30x: Supports model EMW3031.
    • w8801: Supports models EMW3088, 5088.

Enter step

  1. Take the mxosKit development board as an example, connect the “Debug Serial Port” (ie USB power socket) to the PC via “Micro-USB cable”.
  2. Connect the development board "User Serial Port (UART in Aduino Interface)" to the PC via the "USB to TTL Level Module" board.
  3. Open the SecureCRT serial port debugging tool software and find the COM port number corresponding to the “user serial port” in the device manager of the PC.
  4. Create a “user serial port” quick connection in SecureCRT. For the serial port parameter configuration, please refer to: Type: Serial, “user serial port” COM port, baud rate: 921600, data bit: 8bit, parity: none, stop bit :1. The parameters are as follows:

  1. For the mxosKit development board of BOOT and STATUS pin switch, set the DIP switch status of the development board, BOOT: ON (low level), STATUS: OFF (high level), then press "Reset" Press the button to reset the development board.
  2. For the mxosKit development board with the BOOT and STATUS pins connected to the push button switch, press and hold the BOOT button, then press RESET to enter the bootloader mode.

  2. Refer to Bootloader mode to enter the step and enter Bootloader mode.
  3. Before entering the command, you need to change the setting: Session Options → Connection → Serial → Flow Control to cancel the previously selected RTS/CTS; then enter: 0, press Enter, as shown below;

  1. Select Transfer → Send Ymodem in SecureCRT → select the updated Bootloader firmware;

  1. After confirmation, the firmware burning interface is displayed as shown in the figure.

At this moment, the bootloader firmware update is complete.

Read MXOS setting parameters

Command 4: Read or erase/update Flash content

Read the contents of the specified Flash area

  1. Refer to the Bootloader mode to enter the step and enter the Bootloader mode.
  2. Before entering the command, you need to change the setting Session Options → Connection → Serial → Flow Control to cancel the previously selected RTS/CTS; then enter: 4 –dev 0 -r –start 0x800C000 –end 0x800C100, press Enter;
  3. Select Transfer→Receive Ymodem in SecureCRT, and the read file will be stored in the specified directory of SecureCRT.

2. Erase the specified Flash area content

  1. Refer to the Bootloader mode to enter the step and enter the Bootloader mode.
  2. Before entering the command, you need to change the setting Session Options → Connection → Serial → Flow Control to cancel the original selected RTS/CTS; then enter: 4 –dev 0 –e –start 0x0000100 –end 0x0001000, enter; log As shown.
